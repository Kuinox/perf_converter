# PerfToPerfetto.Tests

This test project provides validation helpers for testing PerfToPerfetto output using `trace_processor_shell`.

## Overview

PerfToPerfetto converts Linux perf data (stored in Parquet format) to Fuchsia Trace Format (.ftf) files. This test project provides tools to validate that the conversion process produces correct and well-formed trace files.

## TraceValidationHelper

The main helper class `TraceValidationHelper` provides methods to:

1. **Validate trace files**: Check if a .ftf file can be loaded by trace_processor_shell
2. **Execute queries**: Run SQL queries against trace files to extract information
3. **Get statistics**: Retrieve basic statistics about trace contents (slice count, thread count, etc.)

## Prerequisites

To use the validation helpers, you need:

1. **trace_processor_shell**: The helper will automatically try to find it in common locations or PATH
2. **Valid .ftf files**: Generated by PerfToPerfetto from Parquet input data

## Usage Examples

### Basic Validation

```csharp
var validator = new TraceValidationHelper();
var result = await validator.ValidateTraceFileAsync("output.ftf");

if (result.IsValid)
{
    Console.WriteLine("Trace file is valid!");
}
else
{
    Console.WriteLine($"Validation failed: {result.ErrorMessage}");
}
```

### Getting Statistics

```csharp
var validator = new TraceValidationHelper();
var stats = await validator.GetTraceStatisticsAsync("output.ftf");

Console.WriteLine($"Slices: {stats.SliceCount}");
Console.WriteLine($"Threads: {stats.ThreadCount}");
Console.WriteLine($"Processes: {stats.ProcessCount}");
Console.WriteLine($"Duration: {stats.DurationMs:F2} ms");
```

### Custom Queries

```csharp
var validator = new TraceValidationHelper();
var result = await validator.ExecuteQueryAsync("output.ftf", 
    "SELECT name, COUNT(*) FROM slice GROUP BY name ORDER BY COUNT(*) DESC LIMIT 10;");

if (result.IsSuccessful)
{
    Console.WriteLine("Top slice names:");
    Console.WriteLine(result.Output);
}
```

### Comprehensive Validation

```csharp
var report = await ValidationSamples.ValidatePerfToPerfettoOutput(
    inputParquetDir: "parquet_output",
    outputTraceFile: "converted.ftf");

if (report.IsValid)
{
    Console.WriteLine($"Conversion successful! Generated {report.Statistics.SliceCount} slices");
}
else
{
    Console.WriteLine($"Validation failed: {report.ErrorMessage}");
}
```

## ValidationSamples Class

The `ValidationSamples` class provides complete examples of how to:

- Validate single trace files
- Analyze trace file contents
- Run custom queries against traces
- Perform comprehensive validation of PerfToPerfetto output

## Installing trace_processor_shell

The validation helper depends on `trace_processor_shell` from the Perfetto project:

1. Download from [Perfetto releases](https://github.com/google/perfetto/releases)
2. Or build from source following [Perfetto build instructions](https://perfetto.dev/docs/contributing/build-instructions)
3. Ensure it's in your PATH or specify the path when creating `TraceValidationHelper`

## Notes

- The helper includes placeholder FuchsiaTraceFormat writer implementations to make PerfToPerfetto buildable
- The actual FTF format implementation would need to be added for production use
- This project focuses on validation infrastructure, not the actual tests themselves
